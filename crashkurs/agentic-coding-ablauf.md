# Agentic-Coding-Crashkurs

## Ziel

Das Team soll schnell verstehen, wie man Coding-Agenten sinnvoll einsetzt:
kleine Aufgaben, guter Kontext, klare Grenzen, Ergebnis pruefen.

## Kurzablauf

1. Was ist Agentic Coding?
2. Unterschied: Chatten vs. Aufgabe delegieren.
3. Gute Aufgabe formulieren: Ziel, Kontext, Grenzen, Akzeptanzkriterien.
4. Agent zuerst lesen lassen.
5. Kleine Aenderung machen lassen.
6. Ergebnis testen und reviewen.
7. Fehler nachbessern lassen.
8. Nicht blind mergen.

## Demo-Idee

Am Hackathon-Projekt zeigen:

- Feature kurz beschreiben.
- Agent laesst sich Kontext geben.
- Agent schreibt Spezifikation oder kleinen Code.
- Mensch prueft.
- Agent verbessert nach Feedback.
